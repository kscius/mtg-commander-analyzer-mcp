# Tribal Strategy Guide

## Key principles

- **Shared creature type** (or “choose a type”) with **lords** and **synergy**.
- **Density**: enough tribal creatures to justify lords.
- **Interaction** and **ramp** still required — tribal is not only creatures.

## Recommended ratios

| Role | Count | Notes |
|------|-------|-------|
| Lands | 35–37 | |
| Ramp | 9–11 | |
| Tribal creatures | 28–40 | Depends on commander |
| Lords / anthems | 6–10 | |
| Removal | 5–7 | |
| Draw | 8–10 | Often creature-based |
| Protection | 2–4 | Protect lord or commander |

## Core card types

- **Lords**: Coat of Arms, Vanquisher’s Banner, type-specific lords
- **Tutors**: only within Bracket tutor caps
- **Changelings**: if supported in colors

## Anti-patterns

- **Too few** creatures of the tribe (lords are dead).
- **Non-tribal** goodstuff breaking density.
- **Board wipes** without recovery (unless recursion theme).
- Mixing **two tribes** without commander support.

## Synergy packages

- **Lord stack**: multiple lords + haste enabler
- **Tribal tutor + lord**: find lord then deploy
- **Commander + typal payoffs**: commander is lord or reduces cost

## Ideal curve

- Curve depends on tribe (elves low, dragons high)
- Adjust ramp to match average CMC

## Commander impact

- Commander **creature type** or **cost reduction** defines which tribe.
- Use `get_synergies` — some commanders use `tribal` or a specific EDHREC theme.

**EDHREC slug:** `tribal`
